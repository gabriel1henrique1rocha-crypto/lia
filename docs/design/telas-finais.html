LIA · Design System

Componentes base — HTML/CSS

Marcação e estilos dos quatro componentes de fundamento (botão, campo, link e cartão), construídos apenas com as variáveis de token. Cada bloco abaixo mostra os estados e traz o HTML pronto para copiar. Use Tab para ver o anel de foco visível.

Botão · .lia-btn

Variantes primária (bordô), secundária (contorno) e fantasma. Tamanho médio garante o alvo de toque de 44px. Estados: repouso, hover, foco (Tab) e desabilitado.

Variantes — tamanho md
Tamanhos — sm · md · lg
Estados

O estado desabilitado usa disabled + aria-disabled="true" (opacidade 0.5, cursor bloqueado).

HTML 
<button type="button" class="lia-btn lia-btn--primary lia-btn--md">Publicar resenha</button>
<button type="button" class="lia-btn lia-btn--secondary lia-btn--md">Buscar livros</button>
<button type="button" class="lia-btn lia-btn--ghost lia-btn--md">Cancelar</button>

<!-- Desabilitado -->
<button type="button" class="lia-btn lia-btn--primary lia-btn--md"
        disabled aria-disabled="true">Desabilitado</button>

<!-- Como link (vira <a>) -->
<a href="/buscar" class="lia-btn lia-btn--secondary lia-btn--md">Buscar livros</a>

Campo · .lia-field

Rótulo ligado ao controle por for/id; a mensagem de erro é ligada por aria-describedby, anunciada com role="alert" e marcada com aria-invalid. O erro é comunicado por ícone + texto, nunca só por cor.

Como aparece na capa da edição.
Informe um e-mail válido (ex.: voce@exemplo.com).
HTML — campo com erro acessível 
<div class="lia-field lia-field--error">
  <label class="lia-field__label" for="email">
    E-mail <span class="lia-field__req" aria-hidden="true">*</span>
  </label>
  <input class="lia-field__control" id="email" type="email"
         aria-invalid="true" aria-describedby="email-error"
         required aria-required="true">
  <span class="lia-field__error" id="email-error" role="alert">
    <svg class="lia-field__error-icon" width="16" height="16"
         viewBox="0 0 16 16" fill="none" aria-hidden="true">
      <circle cx="8" cy="8" r="7" fill="currentColor"/>
      <rect x="7.1" y="4" width="1.8" height="5" rx="0.9" fill="var(--paper-0)"/>
      <circle cx="8" cy="11.4" r="1" fill="var(--paper-0)"/>
    </svg>
    <span>Informe um e-mail válido (ex.: voce@exemplo.com).</span>
  </span>
</div>

Link · .lia-link

Sempre sublinhado para que o significado não dependa só da cor (WCAG 1.4.1). Hover revela o sublinhado pleno; foco mostra o anel azul. Variante discreta para metadados; link externo recebe o indicador ↗.

Cartão · .lia-card

Contêiner editorial. Com href o cartão inteiro vira <a> e ganha realce no hover (sombra + leve elevação) e anel de foco. Subpartes: mídia 3:2, olho, título, resumo e rodapé.
